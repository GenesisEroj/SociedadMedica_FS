Sociedad Médica Altamira
Evaluación Final Transversal – DSY1104 Desarrollo Fullstack II
1. Descripción General del Proyecto

Sociedad Médica Altamira es una aplicación web Full Stack desarrollada como caso semestral para la asignatura Desarrollo Fullstack II (DSY1104).
El sistema fue diseñado para gestionar usuarios y reservas médicas, integrando frontend y backend mediante una arquitectura basada en servicios REST, con autenticación segura y persistencia de datos.

El proyecto integra todas las correcciones realizadas durante el semestre y cumple con los requerimientos técnicos definidos para la Evaluación Final Transversal, tanto en la entrega por encargo como en la presentación y defensa individual.

7. Desarrollo del Backend con Framework y Base de Datos

Indicador IE3.1.1 – Muy buen desempeño

El backend fue desarrollado utilizando Spring Boot, framework backend robusto, con conexión a una base de datos relacional.

El sistema backend incluye:

Modelado de entidades según los requerimientos del cliente

Persistencia de datos mediante JPA/Hibernate

Implementación de la lógica de negocio

Manejo correcto de solicitudes y respuestas HTTP

La aplicación backend resuelve de forma correcta los problemas planteados por los requerimientos funcionales del sistema.

8. Integración Backend – Frontend mediante Servicios REST

Indicador IE3.2.1 – Muy buen desempeño

La integración entre frontend y backend se realizó mediante comunicación REST, permitiendo operaciones completas de tipo CRUD:

Creación de registros

Lectura de información

Actualización de datos

Eliminación de registros

Los endpoints del backend están correctamente implementados y el frontend consume estos servicios de manera efectiva, garantizando la comunicación total entre ambas capas del sistema.

9. Autenticación y Autorización Segura con JWT

Indicador IE3.3.1 – Muy buen desempeño

El sistema implementa autenticación y autorización segura mediante JWT, permitiendo:

Generación de token al iniciar sesión

Validación del token en cada petición protegida

Restricción de acceso a funcionalidades según el rol del usuario

Mantención de la sesión activa incluso al recargar la página

Este mecanismo protege los datos del cliente y garantiza un acceso seguro y controlado a la aplicación.
